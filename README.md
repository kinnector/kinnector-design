# Kinnector Design

Kinnector Design is the centralized repository for UI/UX specifications, style guides, color tokens, and design system variables used across the Kinnector platforms (Desktop EDR, Cloud Dashboard, and Landing Portals).

---

## Why this exists

A cohesive user experience requires absolute visual consistency across desktop dashboards, cloud interfaces, and public portals. Ad-hoc CSS styling creates design regressions, duplicate assets, and slows frontend implementation.

Kinnector Design solves this by providing a unified, tokenized design system for typography, colors, layout structures, and transitions.

---

## Key Assets and Specifications

* **[DESIGN-SYSTEM.md](file:///home/user/Documents/kinnector/kinnector-design/DESIGN-SYSTEM.md)**: Color tokens (dark modes, gradients, HSL tailoring), typography rules (Outfit and Inter Google Fonts), interactive states, and micro-animation specs.
* **[DESIGN-NOVEL.md](file:///home/user/Documents/kinnector/kinnector-design/DESIGN-NOVEL.md)**: Immersion walkthroughs and story-driven user flow definitions.

---

## Core Styling Principles

1. **Tokenized Visuals**: Defines Tailwind, CSS custom variables, and HSL colors directly to ensure a consistent dark mode and visual identity.
2. **Interactive Micro-Animations**: Specifications for interactive states, hover transitions, and visual responses that feel premium.
3. **Typography Standards**: Font family definitions and scale ratios for headers, code blocks, and body copies, replacing browser defaults.
